# Feature Specification: 台指期做空——空頭階梯與空方部位管理（Short Side, Futures-Only）

**Feature Branch**: `003-short-side`

**Created**: 2026-07-07

**Status**: **Re-scoped & Active（2026-07-16）— 台指期期貨限定；前置 008a（資料層）+ 008b（成本/口數）已併入 main，原訂重啟條件全數滿足**

**Input**: 原 OpenSpec 描述對稱的多空階梯（空頭 BOS 下移階梯、看漲 MSS 反手、
空方吊燈止損），但現行 `PositionManager` 僅處理 `direction == 1`（多頭），
`check_entry_signal` 僅接受看漲訊號。2026-07-16 依 brainstorming 定案設計
全面改寫為台指期期貨語意。

> **決策記錄（2026-07-11，採選項 B——治理歷史，保留）**：
>
> 1. **經濟面**：現行五檔標的皆為現貨 ETF／個股，實際做空需借券／信用交易
>    （借券費、券源、平盤下限制）；以 00632R 反向 ETF 替代則有每日再平衡
>    損耗——趨勢系統持有數週的鏡像回測會系統性高估實際可得績效。
> 2. **防禦已內建**：三關價濾網在價格低於中關價時不做多——空頭市況的
>    正確動作（空手）已是系統行為；且現行多頭產能遠未用滿
>    （組合曝險時間個位數百分比）。
> 3. **工程面**：對稱分支使剛完成偏誤清理的引擎測試面翻倍；空方摩擦成本
>    （借券費率）難以取得可靠數字，違反憲法「真實摩擦成本」條款的風險高。
>
> **重啟條件**：當系統支援台指期之資料管線與成本模型（保證金／期貨稅率）時
> 重啟本規格，並將範圍限定於期貨（原生做空、無借券與反向 ETF 損耗問題）。

> **重啟記錄（2026-07-12，採選項 B——治理歷史，保留）**：
> 因 spec 007（MSS 進場區別化）需雙向 MSS 反轉進場，使用者決定重啟本規格並
> 遵守原訂重啟條件：範圍限定**台指期期貨**、不在現貨標的做空。當時前置依賴
> （台指期資料管線與成本模型）未完成，狀態為 Reopened-Blocked。

> **重啟條件滿足記錄（2026-07-16）**：
> **008a**（台指期資料層抽象，PR #13）與 **008b**（期貨成本/口數模型，
> `specs/009-taifex-cost-model`，PR #14）皆已併入 main——重啟條件全數滿足，
> 本規格解除 Blocked、進入完整 Spec Kit 流程。008b 已提供：TAIFEX 權威每口
> 成本（天然對稱、無借券概念）、保證金式整數口 sizing（無方向性）、爆倉護欄。
> 原 FR-004 借券費率條款依此刪除。007 短腿（看跌 MSS 反轉做空，BLOCKED-003）
> 由本規格解封。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 空頭趨勢的對稱訊號與空方回測（Priority: P1）🎯 MVP

研究者對台指期 instrument 啟用做空（`enable_short`）後，空頭市況（價格低於
中關價）能產生**空方進場**並完成可交易回測：空頭 BOS 續勢（下破確認 swing low）
與看跌 MSS 反轉（上升趨勢中跌破確認 swing low，spec 007 短腿）兩條進場路徑，
四維度確認採多方之完整鏡像（動能=收陰線、趨勢=價低於當日開盤與 VWAP、
波動與結構對稱）。空方成本與口數沿用 008b 期貨模型（每口定額+期交稅兩邊+
tick 滑價；保證金式整數口），**無借券費概念**。

**Why this priority**: 台股多頭年約佔六成；缺空方邏輯代表約四成市況無訊號
覆蓋。這是本 spec 的存在理由，也是 007 短腿解封的執行面。

**Independent Test**: mock 台指期資料（含下跌段）回測產生 ≥ 1 筆空方交易，
成本非零、口數為整數、績效可歸因且具決定性。

**Acceptance Scenarios**:

1. **Given** 價格低於中關價且觸發空頭 BOS（下破確認 swing low），**When** 鏡像四維度確認通過（收陰線、價 < 當日開盤且 < VWAP、波動與結構對稱、空方市況濾網通過），**Then** 產生空方進場訊號，於次根開盤成交（賣出開倉）。
2. **Given** 上升趨勢中出現看跌 MSS（跌破確認 swing low，spec 007 語意），且 `enable_short` 開啟，**When** 鏡像反轉 profile 確認通過（放寬順勢確認、全域濾網=價低於中關價、免除市況濾網——多方反轉 profile 之鏡像），**Then** 產生空方反轉進場。
3. **Given** 持有空頭部位，**When** 獲利達 1.5×ATR（價格下跌），**Then** 回補 50%（整數口 floor，0 口時跳過回補）並將止損移至保本位。
4. **Given** 持有空頭部位階段 2，**When** 價格升破空方吊燈線（Rolling Min(Low, n) + m×ATR，只降不升），**Then** 全數回補離場。
5. **Given** 空方交易紀錄，**When** 檢視成本，**Then** 賣出開倉與買回平倉兩邊皆含定額費+期交稅（008b 模型，對稱），無任何借券費項目。

### User Story 2 - 零回歸雙保證（Priority: P2）

`enable_short` 預設關閉。預設下：現貨全部行為與數字**位元不變**；008b 建立的
期貨 long-only 回測數字**位元不變**。現貨標的**結構上不存在**空方路徑——
這不是旗標可開啟的行為，而是資產類別硬邊界（現貨做空之借券/損耗建模
已於 2026-07-11 決策記錄排除）。

**Why this priority**: 橫向硬約束。任何階段違反即回退。

**Independent Test**: 全套既有 pytest 綠；現貨代表標的與 008b 期貨 mock 回測
數字在預設 config 下與現行完全相同；對現貨標的嘗試啟用做空 → 組態驗證
fail-fast。

**Acceptance Scenarios**:

1. **Given** 預設 config（`enable_short` 關閉），**When** 執行現貨與期貨回測，**Then** 全部數字與 008b 完成時的基準逐位元相同。
2. **Given** 現貨 instrument 組態嘗試啟用做空，**When** 載入組態，**Then** 驗證即時失敗並明確報錯（資產類別硬邊界）。

---

### User Story 3 - 多空共存的裁決與監控推播（Priority: P3）

同一期貨標的回測中多空並存：三關價互斥裁決（收盤高於中關價僅考慮多方、
低於中關價僅考慮空方）；持倉中出現反向訊號時，依既有止損/吊燈邏輯先出場、
**不反手**。監控推播（monitor_signals）具備空方能力：迭代期貨 instrument、
檢測空方訊號、多空訊息格式——「能力完備、待真源生效」：期貨現僅 mock 資料源，
mock 下以 dry-run 驗證全鏈路並於訊息標示 mock；真實期貨資料源（後續 spec）
接上即自動生效。

**Why this priority**: 裁決規則是多空共存的正確性前提；推播是使用者選定的
範圍（能力先行），但依賴 mock 標示避免誤導。

**Independent Test**: 裁決單元測試（同根多空訊號→三關價裁決唯一方向）；
推播 dry-run 測試（mock 期貨空方訊號→檢測→格式化→Mock 通知全鏈路，訊息
含 mock 標示與方向）。

**Acceptance Scenarios**:

1. **Given** 同一根 K 線同時滿足多空結構訊號，**When** 進場裁決，**Then** 以三關價裁決唯一方向（高於中關只做多、低於中關只做空），不得同時進場。
2. **Given** 持有多頭部位且出現看跌訊號，**When** 部位管理，**Then** 依既有止損/吊燈先出場，不直接反手（反手不在範圍）。
3. **Given** mock 期貨的空方訊號，**When** monitor_signals 執行，**Then** 訊息含標的、方向（空）、mock 資料標示，並成功送達（Mock 通知端）。

---

### Edge Cases

- **多空訊號同根並存**：三關價全域濾網裁決（互斥：close 與 mid_price 的相對位置唯一決定可考慮方向）。
- **持多遇看跌 MSS／持空遇看漲 MSS**：先依既有止損/吊燈離場，不反手（反手為未來範圍）。
- **空方爆倉由上漲觸發**：空頭部位遇急漲，權益 ≤ 0 當根強制回補結清、模擬終止、標記爆倉（008b FR-011 之方向鏡像）。
- **空方整數口部分回補**：floor(口數×比例)；1 口floor 為 0 → 跳過回補、止損照移保本位（008b FR-012 鏡像）。
- **現貨標的誤設做空**：組態載入 fail-fast（硬邊界），不產生任何空方訊號或交易。
- **空方吊燈只降不升**：追蹤高點回落的鏡像——Rolling Min + m×ATR 隨價格下跌下移、反彈時不上移。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 進場確認 MUST 接受空方結構訊號（`structure_sig == -1`）並以方向對稱的四維度確認：動能端=收陰線；趨勢端=價格 < 當日開盤 且 < VWAP；波動端與結構端與多方對稱。
- **FR-002**: 部位管理 MUST 實作空方分支（`direction == -1`）：止損於價格上穿觸發、獲利 1.5×ATR 回補 50% 並移止損至保本位、階段 2 吊燈採 Rolling Min(Low, n) + m×ATR 且只降不升、時間止盈對稱。
- **FR-003**: 三關價濾網 MUST 對稱且互斥：收盤高於中關價僅允許多方進場、低於中關價僅允許空方進場；同根多空訊號以此裁決唯一方向。
- **FR-004**: 空方摩擦成本與口數 MUST 沿用 008b 期貨模型：賣出開倉與買回平倉兩邊各收定額費+期交稅、滑價以 tick 反映於成交價（賣出開倉向下、買回平倉向上之不利偏移）、保證金式整數口 sizing 與部分回補 floor 語意（008b FR-012）。**不得引入借券費**（期貨原生做空無此概念；原借券費率條款刪除）。
- **FR-005**: 做空 MUST 為期貨限定之明確 opt-in：現貨資產類別**結構上不存在**空方路徑（組態對現貨啟用做空 → 驗證 fail-fast）；期貨經組態旗標 `enable_short` 啟用，**預設關閉**；旗標走 config + Pydantic（憲章 V）。
- **FR-006**: 空方損益 MUST 以方向因子計入權益：損益 = 方向 × 口數 × 點數變化 × 每點價值 − 摩擦成本（方向=-1 時價格下跌獲利）；mark-to-market 權益與爆倉檢查（008b FR-011）同受方向因子約束——空方爆倉由上漲觸發，行為同 008b（當根強制結清、終止、標記）。
- **FR-007**: 空方訊號、sizing 與成交 MUST 遵守看前偏誤紀律（憲章 I）：第 N 根收盤訊號、sizing 用第 N 根收盤時權益、第 N+1 根開盤成交；`tests/test_lookahead_bias.py` MUST 新增空方防禦測試。
- **FR-008**: 看跌 MSS 反轉進場（spec 007 短腿）MUST 解封：`enable_short` 開啟時，看跌 MSS（`mss_signal == -1`）經鏡像反轉 profile（放寬順勢確認、全域濾網=收盤低於中關價、免除市況濾網）觸發空方反轉進場；007 之 BLOCKED-003 註記隨本 spec 完成移除。
- **FR-009**: 空頭 BOS 續勢進場之市況濾網 MUST 方向對稱：趨勢強度（ADX）與噪音（ER）分量無方向共用；長均線分量鏡像（空方=價格低於長均線）。
- **FR-010**: 監控推播 MUST 具備空方能力：迭代期貨 instrument、檢測空方進場訊號、訊息格式含方向；期貨資料源為 mock 時 MUST 於訊息標示 mock（dry-run 語意），真實資料源接上時不需改動即生效。
- **FR-011**: 每條 Success Criterion MUST 對應至少一個自動化測試（憲章 III）。

### Key Entities

- **空方部位**: 方向=-1 之期貨部位——口數（非負整數）、進場價（點）、佔用保證金、止損（上方）、吊燈線（上方、只降不升）、階段狀態；與多方部位互斥（單標的單向）。
- **方向因子**: 損益/權益/爆倉計算之 ±1 因子；多=+1（現行為隱含值）、空=-1。
- **`enable_short` 組態**: 期貨限定之做空開關，預設關閉；現貨設定即組態錯誤。
- **空方訊號**: 空頭 BOS（下破確認 swing low）與看跌 MSS（上升趨勢跌破確認 swing low，007 已產出）；經三關價互斥裁決後成為進場訊號。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001（空方端到端）**: mock 台指期資料回測產生 ≥ 1 筆空方交易：成本非零（兩邊定額+稅）、口數全程非負整數、無借券費項目、結果具決定性。
- **SC-002（鏡像對稱）**: 多空對稱性測試——對映的多/空情境（含進場四維度、部分回補、吊燈、止損）產生一一對應的鏡像行為。
- **SC-003（零回歸雙保證，硬關卡）**: 預設 config（`enable_short` 關閉）下：現貨全套測試與代表標的數字位元不變（008b 基準）；008b 期貨 long-only mock 回測數字位元不變。
- **SC-004（硬邊界）**: 現貨組態啟用做空 → 載入即 fail-fast；現貨路徑任何情況下零空方交易。
- **SC-005（看前偏誤防線）**: 空方進場之訊號/sizing/成交通過末梢截斷不變性測試；成交於訊號次根開盤±滑價 tick（不利方向=賣出開倉向下偏移）。
- **SC-006（空方爆倉）**: 空頭持倉遇急漲致權益 ≤ 0 → 當根強制回補結清、權益曲線截止、標記爆倉。
- **SC-007（007 短腿解封）**: 看跌 MSS 反轉在 `enable_short` 開啟時產生空方進場（mock 資料存在該情境時）；007 spec 之 BLOCKED-003 移除。
- **SC-008（推播 dry-run）**: mock 期貨空方訊號完成檢測→格式化→Mock 通知全鏈路；訊息含方向與 mock 標示。

## Assumptions

- **成本模型完全繼承 008b**：期貨做空與做多成本結構對稱（TAIFEX 定額費+期交稅兩邊、tick 滑價）、保證金與口數公式無方向性——本 spec 不新增任何費率參數；`trading_cost` SoT 不變（憲章 II/V）。
- **反手（stop-and-reverse）不在範圍**：持倉遇反向訊號僅出場。
- **組合回測期貨接入不在範圍**（沿 008b H1 邊界：組合路徑護欄維持）。
- **真實期貨資料源不在範圍**：推播空方為「能力完備、待真源生效」；mock 下 dry-run 並標示。
- **單標的單向**：同一時間單一標的僅一個方向的部位（多空互斥，由三關價裁決 + 不反手保證）。
- **維持保證金追繳不模擬**（沿 008b 明示簡化）：空方同樣僅爆倉護欄 + 止損/使用率緩衝為實質防線。
