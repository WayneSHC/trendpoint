## Clarifications

### Session 2026-07-16

- Q: 爆倉（權益 ≤ 0）時回測該怎麼處理？ → A: 終止模擬——權益 ≤ 0 當根以當根價強制結清、權益曲線截止於該根、summary 如實標記爆倉。
- Q: 期貨整數口下，階段 1「平倉 50%」怎麼執行？ → A: 平倉口數 = floor(持倉口數 × 比例)；floor 為 0（僅 1 口）時跳過部分平倉、但止損仍移至保本位（平倉與風控獨立）。
- Q: 持倉中維持保證金追繳/強制平倉要不要模擬？ → A: 不模擬——僅保留爆倉護欄（FR-011）；階梯/吊燈止損與 50% 使用率緩衝為實質防線；此簡化如實記入 Assumptions。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 期貨誠實摩擦成本回測（Priority: P1）🎯 MVP

研究者對已接入資料管線的台指期 instrument（TXF/MTX/TMF）執行回測，得到**計入真實期貨摩擦成本**的績效：每口定額交易所費用（進、出兩邊各收一次）、期貨交易稅（契約金額之十萬分之二，兩邊各收）、以 tick 計的滑價。008a 的「期貨不可回測」護欄退役——期貨回測不再拋錯，且僅做多（做空訊號屬 spec 003）。

**Why this priority**: 這是本 spec 的存在理由——沒有正確的成本模型，期貨回測數字違反憲章紅線 II（真實摩擦成本），一切後續（003 做空）都不能疊上來。

**Independent Test**: 對 008a 的 mock 期貨資料執行回測：跑通、無拋錯、總摩擦成本 > 0、成本明細與費率公式逐項吻合（單元測試可獨立驗證每一項費用）。

**Acceptance Scenarios**:

1. **Given** 一筆台指期交易（開倉 + 平倉），**When** 計算摩擦成本，**Then** 總成本 = 2 × [每口定額費用 × 口數] + 2 × [契約金額 × 期交稅率] + 2 × [滑價 tick 數 × 每 tick 價值 × 口數]（開、平各一邊）。
2. **Given** TXF 大台（乘數 200）與 MTX 小台（乘數 50）各一筆相同點數的交易，**When** 計算成本，**Then** 兩者的定額費用、稅額、滑價依各自契約規格正確縮放。
3. **Given** 008a 的 mock 期貨 instrument（含拼接連續序列），**When** 執行回測，**Then** 不拋 `FuturesBacktestNotSupportedError`、產生完整績效與交易日誌、所有交易皆為多方。
4. **Given** 期貨回測績效報告，**When** 檢視成本欄位，**Then** 摩擦成本非零且與 config 費率一致（禁止零成本績效，除非明確標註 ablation）。

### User Story 2 - 槓桿部位 sizing 與保證金會計（Priority: P2）

研究者以固定初始資金回測台指期，系統依**保證金**決定口數：每口保證金 = 名目值（點數 × 乘數）× 保證金率；口數 = floor(可用權益 × 保證金使用率 ÷ 每口保證金)，為非負整數。損益以「口數 × 點數變化 × 每點價值」入帳戶權益，報酬率 = 期末權益 ÷ 期初權益 − 1（return-on-margin，槓桿效果自然反映在權益曲線上）。

**Why this priority**: 沒有 sizing 規則，期貨回測無法決定部位大小；保證金式 sizing 使槓桿風險顯性化，是「誠實回測」的另一半。次於 US1 因為成本正確性是紅線、sizing 是交易語意。

**Independent Test**: 單元測試直接驗證保證金公式與口數公式（給定權益、指數、費率參數 → 預期口數）；e2e 驗證回測全程口數為整數、權益曲線無 NaN。

**Acceptance Scenarios**:

1. **Given** 權益 1,000,000、指數 20,000 點、乘數 200、保證金率 5.5%、使用率 50%，**When** 計算口數，**Then** 每口保證金 = 20,000 × 200 × 0.055 = 220,000；口數 = floor(1,000,000 × 0.5 ÷ 220,000) = 2。
2. **Given** 可用權益不足一口保證金，**When** 出現進場訊號，**Then** 不進場（口數 0，不拋錯），並照常繼續模擬。
3. **Given** 持倉期間權益因虧損跌至 0 以下（爆倉），**When** 該根結算，**Then** 以當根價強制結清、模擬終止、權益曲線截止於該根，且 summary 如實標記爆倉（不得產生權益為負仍持續交易的虛構績效）。
4. **Given** 口數 sizing 於第 N 根收盤訊號成立，**When** 決定口數，**Then** 使用第 N 根收盤時點的權益（不得使用第 N+1 根之後的資訊），成交於第 N+1 根開盤（憲章 I）。

### User Story 3 - 現貨路徑位元不變（Priority: P3）

既有現貨（equity）回測的成本、sizing、績效數字**完全不變**：現行 ad-valorem 成本（手續費兩邊、證交稅僅賣邊、比例滑價）與整張 sizing（1000 股）行為逐位元重現。引擎內部改為可組合的成本/sizing 元件不得造成任何現貨回歸。

**Why this priority**: 橫向硬約束（貫穿全程），排 P3 僅因它是「不變」而非「新能力」；任何階段違反即回退。

**Independent Test**: 全套既有 pytest 綠；代表標的回測數字與既有基準逐位元相同（2330.TW 總報酬 15.83%/11 筆、0050.TW 5.43%/13 筆、00878.TW 1.94%/9 筆、00919.TW 10.00%/5 筆；見 spec 007 `baseline-pre-mss.md` 校正後欄）。

**Acceptance Scenarios**:

1. **Given** 現行 config 與現貨標的，**When** 以重構後引擎回測，**Then** 交易數、總報酬、勝率、MDD 與基準逐位元相同。
2. **Given** 既有全套測試（含 spec 004 parity），**When** 執行 `pytest -q`，**Then** 全數通過。

---

### Edge Cases

- **權益不足一口**：口數 floor 為 0 → 略過該進場訊號，不拋錯、不部分進場。
- **爆倉（權益 ≤ 0）**：當根強制結清、模擬終止、summary 標記爆倉；禁止負權益下繼續交易的虛構曲線。
- **微型契約（TMF，乘數 10）**：sizing 粒度極細，公式須對小口值正確（floor 後仍為整數口）。
- **1 口持倉遇階段 1 部分止盈**：floor(1 × 50%) = 0 口 → 不平倉、止損照移保本位（FR-012）；持倉續走至全出場事件。
- **mock 資料 rollover 跳空**：008a 決策——adapter 交付已拼接連續序列，回測直接消費；跳空段視為一般價格波動（真實 back-adjusted 序列不再含 rollover 跳空）。
- **到期現金結算交割費**：不計。本系統之出場（階梯停損／時間出場）皆為到期前平倉，不觸發交割。
- **同 config 混跑現貨與期貨**：各標的依其資產類別取用對應成本/sizing，互不污染。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 每個期貨 instrument MUST 帶契約規格：每點價值（TX=200、MTX=50、TMF=10 NT$/點）、tick 大小（1 點）、交易所每口每邊定額費用（經手費+結算費：TX=20、MTX=12.5、TMF=8.0 NT$）。現貨等效規格為每點價值 1、無定額費用。
- **FR-002**: 期貨每口**每邊**摩擦成本 MUST = 交易所定額費用 + 券商加收定額（預設 0）+ 契約金額 × 期交稅率（0.00002）+ 滑價 tick 數 × 每 tick 價值；開倉、平倉各收一邊。
- **FR-003**: 所有費率與模型參數（券商加收、期交稅率、滑價 tick 數、保證金率、保證金使用率）MUST 以 `config/config.yaml` 之 `trading_cost` 為唯一真實來源並經 Pydantic schema 驗證；契約內生參數（乘數、tick、交易所定額費）隨 instrument 定義集中管理。禁止硬編碼費率（憲章 II、V）。
- **FR-004**: 每口保證金 MUST = 名目值（點數 × 乘數）× 保證金率（config，預設 0.055）。
- **FR-005**: 口數 MUST = floor(可用權益 × 保證金使用率 ÷ 每口保證金)，為非負整數；口數 0 時不進場。
- **FR-006**: 期貨損益 MUST 以「口數 × 點數變化 × 每點價值 − 摩擦成本」計入帳戶權益；報酬率 = 期末權益 ÷ 期初權益 − 1；與現貨共用同一初始資金基底。
- **FR-007**: 口數 sizing MUST 只使用訊號根（第 N 根）收盤時已知之權益；成交於第 N+1 根開盤（憲章 I）；新期貨路徑 MUST 在 `tests/test_lookahead_bias.py` 增加防禦測試。
- **FR-008**: 008a 之期貨回測拒絕護欄 MUST 於**單標的路徑**退役（引擎層 + `run_backtest.py` 入口）：對期貨 instrument 回測不再拋 `FuturesBacktestNotSupportedError`。**組合回測路徑之護欄 MUST 保留**（組合的期貨元件接入不在本 spec 範圍，放行將使期貨掉入現股成本路徑、違反憲章 II）。本 spec 範圍內期貨僅做多（回測紀錄不得出現空單；做空為 spec 003）。
- **FR-009**: 現貨回測路徑 MUST 位元不變：既有成本語意（手續費兩邊、證交稅僅賣邊、比例滑價）、整張 sizing、全部績效數字與現行完全相同。
- **FR-010**: 引擎 MUST 以可替換的成本／sizing／契約元件按資產類別組裝（現貨元件精確重現現況、期貨元件實作本 spec 模型），共用逐根迴圈、訊號與出場邏輯；每條 Success Criterion MUST 對應至少一個自動化測試（憲章 III）。
- **FR-011**: 爆倉防護：權益 ≤ 0 當根 MUST 以當根價強制結清並終止模擬，權益曲線截止於該根，summary 如實標記爆倉。
- **FR-012**: 部分出場之整數口規則：部分平倉口數 MUST = floor(持倉口數 × 出場比例)；結果為 0 口時 MUST 跳過該次部分平倉但仍執行對應風控動作（如止損移至保本位）；持倉口數任何時點 MUST 為非負整數。

### Key Entities

- **ContractSpec（契約規格）**: 期貨契約的內生屬性——每點價值/乘數、tick 大小、交易所每口定額費用。附掛於 instrument 定義（008a 之 `Instrument`），不可變。
- **成本模型（CostModel）**: 按資產類別計算每筆成交的摩擦成本。現貨版重現 ad-valorem 現況；期貨版實作 FR-002。
- **部位 Sizer（PositionSizer）**: 按資產類別決定部位單位。現貨版重現整張邏輯；期貨版實作 FR-004/FR-005 保證金式口數。
- **期貨交易紀錄**: 交易日誌擴充——口數、進出價（點）、佔用保證金、成本明細（定額費/稅/滑價）、損益。
- **`trading_cost.futures` 組態**: 券商加收（預設 0）、期交稅率（0.00002）、滑價 tick 數（1）、保證金率（0.055）、保證金使用率（0.5）。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001（現貨 parity，硬關卡）**: 全套既有測試通過，且代表標的回測數字與基準逐位元相同（2330.TW 15.83%/11 筆、0050.TW 5.43%/13 筆、00878.TW 1.94%/9 筆、00919.TW 10.00%/5 筆）。
- **SC-002（期貨成本正確）**: 單元測試逐項驗證：每口定額費兩邊、期交稅兩邊（契約金額 × 0.00002）、滑價 tick 計價、TX/MTX/TMF 三契約縮放正確。
- **SC-003（保證金/口數正確）**: 單元測試驗證每口保證金公式與口數 floor 公式（含權益不足 → 0 口、TMF 小口值）。
- **SC-004（期貨端到端）**: mock TXF 與 MTX 資料 → 回測跑通、總摩擦成本 > 0、全程口數為非負整數、權益曲線無 NaN、無空單。
- **SC-005（看前偏誤防線）**: `tests/test_lookahead_bias.py` 新增測試證明：截斷訊號根之後的資料不改變該根的 sizing 決策與成交價；期貨成交於訊號次根開盤。
- **SC-006（護欄退役）**: 單標的路徑對期貨 instrument 回測不拋 `FuturesBacktestNotSupportedError`；**組合路徑對期貨仍拋錯**（範圍邊界護欄）；現貨路徑（預設）行為不受影響。
- **SC-007（費率 SoT）**: 引擎與腳本原始碼中無硬編碼期貨費率／保證金常數（grep 稽核）；全部經 config + Pydantic 取得。

## Assumptions

- **權威費率（TAIFEX 官方「交易及結算相關費率」, https://www.taifex.com.tw/cht/4/feeSchedules ，2026-07 經瀏覽器渲染核實）**：交易所每口每邊定額 = 經手費 + 結算手續費：TX 12+8=20、MTX 7.5+5=12.5、TMF 4.8+3.2=8.0（NT$）；期交稅率 = 契約金額 × 0.00002（十萬分之二），兩邊各收，台指類契約全部適用。到期現金結算之交割手續費不計（本系統到期前平倉）。
- **券商加收預設 0 之建模選擇**：交易所費率為期貨商繳交易所之權威下限；券商向交易人實收另議。預設 0 = 「可辯護的權威下限、非零成本」；使用者可將實際券商每口費率填入 config 以貼近實付。此選擇使預設績效略優於散戶實付，spec 如實註明。
- **保證金率 0.055 為近似**：台指期原始保證金/名目值歷史約 4–6%；以名目值百分比表示可跨年代自動縮放（避免固定 NT$ 之時代錯置），非交易所逐期公告之精確值。
- **return-on-margin 會計與現貨共用初始資金基底**：期貨權益曲線與現貨曲線**不直接可比**（槓桿本質）；保留同一基底是框架一致性選擇，非跨資產績效可比性宣稱。
- **滑價預設 1 tick/邊**（1 tick = 1 指數點）；保守情境可調高，屬 config 參數。
- **不模擬維持保證金追繳/強制平倉**（明示簡化）：真實帳戶在權益跌破維持保證金（約原始 75%）時會被追繳/強平；本模型僅保留爆倉護欄（FR-011），實質防線為階梯/吊燈止損與保證金使用率 50% 緩衝。歷史維持保證金精確值難以取得；如後續（如 spec 003）需要可再加強。
- **資料前提**：期貨價格序列由 008a adapter 交付之已拼接連續序列（rollover 已由資料層處理）；本 spec 不做框架內 rollover 引擎。
- **範圍邊界**：做空、監控推播之期貨支援、真實 TAIFEX/券商資料 adapter、Walk-Forward/優化器之期貨接入，皆不在本 spec（做空 = spec 003；其餘視後續需求）。
