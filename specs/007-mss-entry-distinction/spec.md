# Feature Specification: MSS 進場區別化（fractal 反轉校正）

**Feature Branch**: `007-mss-entry-distinction`

**Created**: 2026-07-12

**Status**: Draft

**Input**: User description（摘要）: 於 spec 002（FVG 確認）實作時發現，MSS 在進場路徑被 BOS 完全涵蓋（`mss ⊆ bos`），導致任何調整 MSS 品質（含 FVG）的努力對回測 P&L 零影響。依 `docs/ladder-optimization-research.md`，MSS 本應是「突破反向關鍵波段點」的**趨勢反轉**訊號、與 BOS「突破同向波段點」的**續勢**訊號有別，但現行程式（`ladder_system.py:163-169`，作者自標「此處簡化示意」）把 MSS 誤實作為「高量能的同向 BOS」。本規格校正 MSS 語意並賦予其獨立的反轉進場。

## Clarifications

### Session 2026-07-12

- Q: 在 Long-Only 產品下，MSS 反轉進場該怎麼行為？ → A: 重啟雙向——推翻 spec 003（2026-07-11）的 Long-Only 產品決策；MSS 雙向反轉（看漲反轉做多、看跌反轉做空），維持「先平再開」。
- Q: 重啟短側要怎麼結構化？ → A: 重開 spec 003（短側）為獨立前置依賴，先補齊做空的部位/出場/成本/測試；007 專注雙向 MSS fractal 反轉，並明標依賴 003。

## User Scenarios & Testing *(mandatory)*

本工具的使用者為量化研究者/交易者（Wayne）。以下三個使用者旅程各為獨立可測、可獨立交付價值的切片，依重要度排序。

### User Story 1 - 校正 MSS 為理論的反轉語意（Priority: P1）

作為研究者，我要 MSS 訊號真正代表「趨勢結構被反向突破」（跌破上升結構的近期關鍵支撐、或突破下降結構的近期關鍵壓力），而非現行的「高量能同向突破」，使 MSS 與 BOS 成為語意分離的兩種訊號，MSS 不再是 BOS 的子集。

**Why this priority**: 這是整個問題的根因修正，也是後續一切（反轉進場、FVG 回接）的地基。單獨交付即有價值——即時監控的「反轉告警」語意會立刻正確化。

**Independent Test**: 在代表性歷史資料上，能找到至少一個 bar 其 MSS 訊號為 ±1 而同向 BOS 不成立；並以真值表單元測試覆蓋 HH/HL/LH/LL × 突破/未突破 × 位移有無；看前偏誤防禦測試通過。

**Acceptance Scenarios**:

1. **Given** 一段處於上升結構（近期呈 HH/HL 序列）的 K 線，**When** 收盤價跌破最近一個「已確認」的較高低點（HL）且伴隨位移確認，**Then** 系統於該 bar 產生看跌 MSS（-1）。
2. **Given** 一段處於下降結構（近期呈 LH/LL 序列）的 K 線，**When** 收盤價突破最近一個「已確認」的較低高點（LH）且伴隨位移確認，**Then** 系統於該 bar 產生看漲 MSS（+1）。
3. **Given** 一個「同向續勢突破」（上升結構中創新高），**When** 未發生反向波段點突破，**Then** 該 bar 為 BOS 而非 MSS（兩者互斥）。
4. **Given** 一個反向波段點被突破但無位移確認（僅價格穿越、量能不足），**When** 判定 MSS，**Then** 不產生 MSS 訊號。

---

### User Story 2 - MSS 獨立反轉進場，讓 MSS 品質影響回測 P&L（Priority: P2）

作為研究者，我要校正後的 MSS 觸發一條**獨立的反轉進場**，與既有 BOS 續勢進場並存，使「調整 MSS 品質」的努力能真正反映在回測交易與 P&L 上（對照 spec 002 T014 的零 delta）。

**Why this priority**: 這是讓 MSS 從「進場路徑虛設」變為「有作用」的關鍵切片，直接兌現本規格的核心動機。

**Independent Test**: 以同一資料、同一成本假設跑「校正前 vs 校正後」回測，交易數/勝率/EV/MDD 至少一項出現可量測的非零差異。

**Acceptance Scenarios**:

1. **Given** 系統無部位，**When** 產生 MSS 反轉訊號且通過既有市況/濾網，**Then** 依 MSS 方向開立新部位。
2. **Given** 系統持有與 MSS 反向的部位，**When** 產生 MSS 反轉訊號，**Then** 先平掉既有部位再依 MSS 方向反向開倉（先平再開）。
3. **Given** 系統持有與 MSS 同向的部位，**When** 產生 MSS 反轉訊號，**Then** 不重複進場。
4. **Given** 既有 BOS 續勢進場情境不變，**When** 只有 BOS 訊號、無 MSS，**Then** 進場行為與校正前一致（BOS 路徑無回歸）。

---

### User Story 3 - FVG 確認經校正後的 MSS 對回測 P&L 產生作用（Priority: P3）

作為研究者，我要驗證 spec 002 的 FVG 確認閘門套用在校正後的 MSS 之後，開/關 FVG 會對回測 P&L 造成可量測差異，回應 spec 002 SC-002 留下的「FVG 對回測 EV 貢獻為零」懸案。

**Why this priority**: 這是對前一個懸案的收尾驗證；價值明確但依賴 US1、US2 已落地，故排 P3。

**Independent Test**: 在校正後系統上跑 FVG on 與 FVG off 兩組回測，比較 P&L/交易數差異。

**Acceptance Scenarios**:

1. **Given** 校正後系統與 FVG 關閉的回測基準，**When** 開啟 FVG 確認，**Then** MSS 訊號數與/或回測 P&L 出現可量測差異。
2. **Given** FVG 開啟，**When** MSS 反向突破發生但近設定根數內無同向 FVG，**Then** 該 MSS 訊號被歸零、不觸發進場。

---

### Edge Cases

- **資料起點不足**：樞紐點需其後 N 根方能確認，資料前段（約前 2N 根）無法產生穩定結構與 MSS；此區間 MSS 一律為 0，不得報錯。
- **無明確趨勢結構（盤整）**：缺乏可判定的 HH/HL 或 LH/LL 序列時，MSS 不觸發（沒有「被反向突破的結構」可言）。
- **同一 bar 同時符合 BOS 與 MSS 條件**：設計上以趨勢偏向使兩者互斥；規格要求明確定義並以測試鎖定，不得出現同 bar 同向同時 BOS 與 MSS 皆成立。
- **反轉訊號但無既有部位**：直接依方向開新倉，無「先平」動作。
- **反轉訊號但已持同向部位**：不重複進場。
- **位移確認不足**：反向波段點被穿越但量能/位移不足 → 不成立 MSS。
- **FVG 閘門開啟且近根無同向 FVG**：MSS 歸零。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 以對稱碎形偵測波段高/低點（swing high/low），且任一樞紐點 MUST 僅在其後 N 根 K 線收盤後方可被訊號判斷引用（看前偏誤防禦）。
- **FR-002**: 系統 MUST 由「已確認」樞紐序列判定市場結構分類（HH/HL/LH/LL）與當下趨勢偏向。
- **FR-003**: 系統 MUST 將看跌 MSS 定義為「上升結構中收盤跌破最近已確認 HL 且伴隨位移確認」，看漲 MSS 對稱定義為「下降結構中收盤突破最近已確認 LH 且伴隨位移確認」。
- **FR-004**: BOS 續勢語意 MUST 維持不變（突破同向波段點）；校正後 MSS 與 BOS MUST NOT 為子集關係——MUST 存在「MSS 訊號成立而同向 BOS 不成立」的 bar，且同一 bar 同向不得同時判為 BOS 與 MSS。
- **FR-005**: 系統 MUST 提供獨立的 MSS 反轉進場路徑，與既有 BOS 續勢進場並存；既有 Chandelier 出場邏輯 MUST NOT 被本規格更動。
- **FR-006**: MSS 反轉進場觸發時，若持有反向部位 MUST 先平倉再反向開倉；若無部位 MUST 直接開倉；若已持同向部位 MUST NOT 重複進場。回測交易紀錄中 MUST NOT 出現同時持有多空的情形。
- **FR-007**: MSS 進場的追價積極度（k 值）MUST 可組態，且預設 MUST 等於既有 BOS 進場 k 值。
- **FR-008**: spec 002 的 FVG 確認閘門 MUST 套用於校正後的 MSS；FVG 開啟時，MSS MUST 於近設定根數內存在同向 FVG 方成立，否則歸零；FVG 關閉時輸出 MUST 與未套 FVG 之校正後基準一致。
- **FR-009**: 所有新增可調參數（碎形強度 N、反轉進場開關、MSS 進場 k、位移門檻）MUST 集中於中央組態並經 schema 驗證，MUST NOT 硬編碼於演算法、回測或 UI。
- **FR-010**: 所有回測與績效對照 MUST 計入手續費、證交稅與滑價，費率以中央組態之交易成本區塊為唯一來源。
- **FR-011**: 系統 MUST 為校正後的 MSS 新增看前偏誤防禦測試——對任一 bar t，其 MSS 訊號在遮蔽 t 之後（含 `(t−N, t]` 尚未確認之樞紐）的資料後保持不變——並延續「第 i 列僅依賴前 i 列」的時序契約測試。
- **FR-012**: 本規格每一條驗收標準 MUST 對應至少一個 pytest 測試；無法自動化者 MUST 於規格標註 `[MANUAL]` 並說明人工驗證步驟。

### Key Entities

- **Swing Point（樞紐點）**：某 bar 的高（或低）為對稱窗口 `[i−N, i+N]` 之極值。屬性：位置、值、型別（high/low）、確認時點（bar `i+N`）。
- **Market Structure（市場結構）**：由已確認樞紐序列分類出的 HH/HL/LH/LL 與趨勢偏向（上升/下降/不明）。
- **MSS 反轉訊號**：方向（+1/−1）、觸發 bar、被突破的反向波段點參照、位移是否確認、FVG 是否確認。
- **MSS 反轉進場動作**：進場方向、對既有部位的處置（先平再開／開新／略過）、使用的進場 k 值。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 在代表性歷史資料上，存在至少一個 bar 其 MSS 訊號為 ±1 而同向 BOS 不成立（量化證明 MSS 不再是 BOS 子集）。
- **SC-002**: 校正後相對校正前的回測（同一資料、同一成本假設），在交易數、勝率、期望值(EV)、最大回撤(MDD) 中至少一項出現可量測的非零差異（與 spec 002 T014 零 delta 成明確對照）。
- **SC-003**: 在校正後系統上，FVG 確認開/關的回測 P&L 出現可量測差異（回應 spec 002 SC-002 懸案）。
- **SC-004**: 看前偏誤防禦測試 100% 通過：對任一 bar t，遮蔽 t 之後資料後其 MSS 訊號不變。
- **SC-005**: MSS 反轉進場對既有反向部位一律先平再開，回測交易紀錄中不存在同時持有多空的情形。
- **SC-006**: 合併前 `pytest` 全綠。

## Assumptions

- **代表性歷史資料**：沿用既有回測所用之標的與期間（延續 spec 001/002 之基準資料集）。
- **位移確認**：預設沿用現行量能 proxy（成交量 > 20 週期均量的 1.5 倍）作為位移確認，保留後續升級為 ATR 正規化 range 的空間。
- **碎形強度 N**：預設 2（Williams 標準的五根碎形）；N 同時決定樞紐確認延遲為 N 根。可組態。
- **MSS 進場 k**：預設等於既有 BOS 進場 k（先證明機制方向性，k 值優化留待後續規格）。
- **趨勢偏向來源**：可重用系統既有市況/方向資訊判定趨勢偏向；確切機制於 `/speckit-plan` 定案。
- **建議組態鍵名**（`swing_fractal_n`、`mss_reversal_entry`、`mss_ladder_k`、位移門檻參數）為建議，最終命名於 `/speckit-plan` 定案。
- **執行語意**：訊號於第 N 根收盤產生、第 N+1 根開盤成交（延續憲章 I）。
- **前置依賴（spec 003，2026-07-12 重開，台指期限定）**：本規格為雙向（多+空）。短側能力（做空部位/出場/成本/測試）由重開的 spec 003 承接，但 003 範圍限定**台指期期貨**且阻塞於台指期資料管線 + 成本模型（見 spec 003 §重啟記錄；理由為規避紅線 II 摩擦成本）。因此 007 採**長側先行**：長側 MSS 反轉進場即可達成 SC-001/003/004 與長側非零 delta；看跌反轉「做空」進場阻塞於 003 → 台指期基礎建設（現行現貨程式為 Long-Only，僅買進做多）。
- **反轉進場閘門**：反轉進場不得被趨勢跟隨型市況濾網（如 200MA 長期均線方向）靜默封殺——看漲反轉本質發生在均線下方、看跌反轉發生在均線上方。反轉進場所用的市況閘門（放寬/改用趨勢強度 ADX 與災難濾網，而非順勢方向濾網）之確切設計於 `/speckit-plan` 定案。

## Out of Scope

- **重啟短側機制本身**（做空部位管理、做空出場、做空成本模型與相關測試）不在 007 範圍，由重開的 spec 003 負責；007 僅在其上建立雙向 MSS 反轉進場並明標依賴。
- 不更動既有 Chandelier 出場邏輯（本規格只動進場側；短側 Chandelier 出場由 spec 003 補齊）。
- 不重寫 `monitor_signals.py` 告警邏輯（MSS 語意校正後告警會自然更精準，但不重構告警）。
- MSS 進場 k 值的最佳化（本規格預設 `mss_ladder_k = ladder_k`，優化留待後續規格）。
- 位移確認由量能 proxy 升級為 ATR 正規化（保留空間，非本規格必辦）。
