# 制度維護協議（Maintenance Protocol）

這套制度檔（CLAUDE.md + `.claude/docs/*.md` + 記憶庫）由後續 session 的模型維護。
本檔規定：什麼可以自己改、什麼要先問、教訓寫回哪裡、什麼格式、多長要精簡。

## 1. 改動分級

**可自行改（不必問使用者，但要在 commit message 或回覆中告知）**：
- 修正事實錯誤：失效的路徑、改名的檔案、不存在的工具/參數名。
- 新增：踩坑教訓（見第 3 節）、模板的填空欄位、rubric 的正反例。
- 精簡：合併重複規則、刪除已註明過時的內容（保留備份，見第 2 節）。

**先問使用者才能改**：
- 刪除或反轉一條既有規則（規則存在通常有你看不到的歷史原因；
  例外：該規則引用的東西已不存在且無等價物，可改，但要留註記）。
- 改動指令優先序、鐵律清單（CLAUDE.md「鐵律」節）、誠實條款。
- 任何使用者層設定：`~/.claude/settings.json`、claude.ai connector、
  搬移 repo 位置、授權條款變更。

**永遠不准**：把制度檔內容默默改弱以讓自己當下的產出「合規」。
驗收條件是用來過的，不是用來改的（`judgment-rubrics.md` 第 4(d) 條同源）。

## 2. 改前備份

修改任何既有制度檔前：先 `cp` 一份到 `.claude/docs/backups/{檔名}.{YYYY-MM-DD}.md`
（目錄不存在就建立）。新增全新檔案不需備份。同一原始檔的備份超過 5 份時，
刪該檔最舊的一份。
repo 有 git，但備份檔另存的原因：制度檔的改壞常在數個 session 後才被發現，
屆時 git 歷史已混入大量無關 commit，備份檔讓回滾不需要考古。

## 3. 踩坑教訓寫回哪裡（決策樹）

踩坑後，趁 session 還記得，按這個順序判斷寫到哪：

1. **跟這個 repo 的程式碼/工作流綁定**（例如：某測試在無網路環境必掛、
   某指令在 Drive 路徑下要加引號）→ 對應的 `.claude/docs/` 檔案；
   找不到對應檔就寫進 CLAUDE.md 鐵律節「附註」（超過 3 條就開新檔抽走）。
2. **跟使用者或機器環境綁定、跨專案有效**（例如：使用者偏好繁中回覆、
   Google Drive 路徑特性）→ 記憶庫 `~/.claude/projects/...TrendPoint/memory/`，
   一坑一檔，並在 MEMORY.md 加一行索引。
3. **跟單次任務綁定、下次不會再遇到** → 不寫。制度檔的敵人是噪音。

同一主題兩邊都沾時，用「換一個 repo 還成立嗎？」裁決：成立（機器/使用者層事實）
→ 記憶庫；不成立（只對本 repo 的指令與路徑有效）→ `.claude/docs/`。

**格式**（教訓條目一律三行式）：
```
- 坑：{一句話，具體到可辨識}
  為什麼：{根因一句話}
  下次怎麼做：{可執行的動作，含指令或判準}
```

## 4. 精簡節奏（防制度肥大）

- 每次要在制度檔**新增**內容時，先問：有沒有一條舊內容因此變得多餘？有就順手刪（備份後）。
- 硬上限：CLAUDE.md ≤ 100 行；單一 `.claude/docs/` 檔 ≤ 150 行。超標時的精簡順序：
  (1) 合併重複、(2) 例子每條判準最多留一正一反、(3) 把低頻內容抽成新引用檔。
- 過時標記：拿不準能不能刪的內容，先標 `[過時候選 YYYY-MM-DD]`；
  之後任兩次有 session 依本節執行精簡動作時，該標記仍沒人喊冤就刪。

## 5. 制度變更的驗收

改完制度檔後，用 `judgment-rubrics.md` 第 5 節的文件底線驗法：
派一個 fresh-context agent（sonnet 即可）只讀改後的檔案，要求它複述被改的規則
並舉一個執行例。複述錯誤 = 你寫得不夠明確，修到它讀得懂為止。
改動 ≤3 行的小修正可免驗，但 read-back 自己看一遍。

## 6. 記憶庫格式（與 harness 記憶機制對齊）

記憶檔放 `~/.claude/projects/-Users-shihhuichi-...TrendPoint/memory/`（實際路徑
以當期 system prompt 宣告為準），frontmatter 必含 `name`（kebab-case）、
`description`（一行摘要）、`metadata.type`（user/feedback/project/reference）。
寫完在同目錄 MEMORY.md 加一行 `- [標題](檔名.md) — 一句話鉤子`。
存之前先檢查是否已有同主題檔案——更新舊檔優於開新檔；發現錯誤記憶就刪除。
相對日期一律轉絕對日期。
注意：若當前 session 在 `.claude/worktrees/` 下工作，記憶一律寫回**主 repo**
對應的 memory 目錄（路徑不含 `--claude-worktrees-` 後綴的那個），
否則教訓會困在 worktree 專屬目錄，主 repo 的未來 session 讀不到。
